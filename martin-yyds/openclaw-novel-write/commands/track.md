# /novel track - 综合追踪

## 触发方式

```
/novel track [选项]
/novel track --check    # 写作后追踪验证
```

或对话式：
```
更新进度追踪
检查写作状态
```

---

## 功能

1. **追踪更新**：更新章节进度、字数统计
2. **一致性检查**：验证角色状态、关系变化
3. **情节追踪**：确认线索埋设/揭晓进度
4. **质量验证**：字数是否达标、要素是否齐全

---

## 执行方式

### 写作后自动执行（after_each_chapter）

每次 `/novel write` 完成后 AI 自动执行 `--check`（失败阻断写作）

### 手动触发

```
/novel track
```

---

## 时间线校验

`/novel track --check` 执行时，必须读取 `stories/<story-name>/timeline.md` 进行时间线校验：

### 校验内容

1. **章节事件 vs 时间线顺序**
   - 本章写到的事件，时间点是否与 timeline.md 中的顺序一致
   - 不能出现"第3年事件"写在"第2年事件"之前

2. **章节事件 vs 时间线覆盖**
   - 本章写到的事件，是否在 timeline.md 中有记录
   - 如果章节内出现了时间线中没有的事件，记录为「时间线遗漏」

3. **预期外事件检测**
   - 章节内描述了时间线没有的因果链或转折
   - 提示用户是否需要更新 timeline.md

### 校验流程

```
读取 timeline.md
    ↓
读取本章 content/volumeX/chapter-XX.md
    ↓
提取本章涉及的时间点和事件
    ↓
对照 timeline.md 逐条校验
    ↓
发现问题 → 阻断 → 提示修复
    ↓
无问题 → 通过
```

### 校验失败示例

```
❌ 时间线校验失败

问题类型：事件顺序矛盾
具体问题：
  - timeline.md 记录：事件002（第1年7月）→ 事件003（第3年5月）
  - 第2章内容：主角在第1年7月提到"三年前的往事"
  - 矛盾：第1年7月的角色不可能提到第3年才发生的事

影响范围：第2章
建议修复：修正章节内容或更新 timeline.md
```

```
⚠️ 时间线遗漏

问题类型：章节事件未在时间线中记录
具体问题：
  - 第4章描述了主角妹妹的生日宴会
  - timeline.md 中无此事件记录

建议：更新 timeline.md，补充事件 007（第2年2月 - 妹妹生日）
```
