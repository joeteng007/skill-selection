# ask — 小智追问框架 v5.0 | 单Agent完整版

> **触发词**：`ask ` 或 `ask：`
> **版本**：v5.0（单Agent完整分析版）
> **核心改进**：去掉sub-agent依赖，单次会话内完成"分析→批评→追问→收敛"全流程

---

## 定位

**ask = 一句话触发的完整研究分析流程**

用户说"ask [问题]"，小智在**单次回复中**完成：
1. 初始分析（龙虾视角）
2. 批评者攻击（批评者视角，主动嵌入）
3. 追问收敛（小智追问 + 龙虾回应）
4. 最终结论（量化+置信度+监测指标）

---

## 执行流程

用户输入：`ask [任何投资/宏观/策略问题]`

小智在**单次回复中**按以下结构输出：

## 追问规则

1. **每轮只追问1个最关键问题**，不问多余的
2. **数字必量化**：没有具体数字的结论，追问到底
3. **来源必标注**：⭐⭐⭐（权威）/ ⭐⭐（机构）/ ⚠️（估算）
4. **收敛判断**：出现以下任意一条→停止追问，进入收敛
   - 核心问题已回答（三情景+概率+触发条件齐全）
   - 追问3轮后无新增实质信息
   - 关键数据无法获取（明确标注⚠️）

---

## 收敛置信度评分标准

| 评分 | 含义 |
|------|------|
| 4/4 | 完整数据+多源验证+逻辑严密 |
| 3/4 | 数据完整但有1-2个假设未验证 |
| 2/4 | 主要论点有数据支撑，但有明显盲点 |
| 1/4 | 结论依赖估算，争议较大 |

---

## 简化原因（对比v4.0）

| v4.0（复杂版） | v5.0（单Agent版） |
|---------------|------------------|
| 依赖sub-agent Spawn | 无需sub-agent，单Prompt完成 |
| 多轮外部回调 | 单次会话内完整输出 |
| 时序控制复杂 | 线性结构，可追踪 |
| 批评者独立运行 | 批评者嵌入主Prompt |

## 适用场景

- ✅ 复杂宏观判断（黄金/美联储/汇率）
